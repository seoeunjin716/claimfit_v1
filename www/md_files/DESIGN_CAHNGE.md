# 사정의 고수 - 3단계 의뢰 신청 폼 디자인 개선 전략 문서  
(수직 레이아웃·여백 재분배 & 전문성 강화)

**작성일** : 2026년 2월  
**작성자** : EUNJIN (with Grok)  
**목적** :  
현재 폼 디자인이 상단에 콘텐츠가 밀집되어 압박감을 주고, ‘지피티스러운’ 단조로움과 유치한 느낌이 남아 있는 문제를 해결.  
전문 손해사정 서비스다운 세련되고 신뢰감 있는 UI로 업그레이드하며, 사용자가 위에서 아래로 자연스럽게 정보를 읽고 입력할 수 있도록 재구성.

## 1. 핵심 문제점 진단

- **밀집도·압박감**  
  - 상단 스텝 바 + 타이틀 + 첫 번째 카드가 화면 상단에 너무 붙어 있음  
  - 카드 내부 요소(라벨·입력창·버튼)가 상단에 쏠려 아래쪽 여백 과다 → 시각적 불균형  

- **지피티스러운 단조로움 & 유치함**  
  - 과도하게 둥근 모서리, 얇은 선 박스, 기본 검은색 텍스트 → AI 생성 느낌 강함  
  - 타이포그래피 위계 부족 → 모든 텍스트가 비슷한 무게감  
  - 입력 필드 포커스 시 변화 없음 → 정적·저렴해 보임  

- **시선 흐름 문제**  
  - 섹션 간 구분 미흡 → 정보가 한 덩어리로 느껴짐  
  - 그룹화 부족 → 사용자가 “다음은 뭐지?” 하며 헤맴  

## 2. 개선 원칙 (Professional & Breathing Space)

- **공기감(Negative Space) 최우선**  
  - 전체 화면에 여백을 적극 부여 → 숨쉬는 느낌  
  - 상단 여백 확대, 카드 내부 균등 패딩  

- **수직·수평 중앙 정렬**  
  - 카드 내부 콘텐츠 vertical center + horizontal center (필요 시 left align 조합)  
  - 텍스트·버튼 모두 중앙 배치로 조화롭게  

- **타이포그래피 위계 강화**  
  - 메인 타이틀 : Bold 24–28px, 다크 그레이 #111827  
  - 서브 설명 : Medium 16px, 중간 그레이 #4B5563  
  - 라벨 : Medium 14–16px, #374151  

- **디테일 폴리싱**  
  - 포커스 애니메이션 : 은은한 브랜드 컬러 테두리 + subtle shadow  
  - 모서리 곡률 : rounded-lg (12–16px) → rounded-md (8–12px)로 톤다운  
  - 색상 : Primary #2563EB 제한 사용, Success #16A34A 미니멀 적용  

## 3. 구체적 적용 가이드 (화면 전체)

### 3.1 전체 레이아웃 재분배
- **상단 여백 확대**  
  - Safe Area 아래 40–56px (기존 16–24px → 1.5~2배)  
  - 스텝 바와 메인 타이틀 사이 : 32px  
  - 타이틀과 첫 번째 카드 사이 : 40px  

- **스텝 바 개선**  
  - 원형 번호와 텍스트 간격 : 12px → 16–20px  
  - 진행 중 단계 : Primary #2563EB + bold  
  - 대기 단계 : Gray-400 + 얇은 선  
  - 전체 높이 : 64px → 72px (숨쉬는 느낌)  

### 3.2 카드 내부 중앙 정렬 & 여백 균등화
- **카드 공통 스타일**  
  - padding: EdgeInsets.all(24) 또는 28px  
  - min-height: 140–180px (콘텐츠 적어도 빈 공간 없음)  
  - 내부 구조 : Column(mainAxisAlignment: MainAxisAlignment.center)  

- **라벨·입력창 배치**  
  - 라벨 : center 또는 left + 입력창 center  
  - 입력창 : full width, 높이 52–56px, focus 시 border #2563EB + shadow sm  
  - 요소 간 margin-bottom : 20–24px (기존 12–16px)  

- **선택 버튼 그룹 (치료 구분 등)**  
  - 모서리 : rounded-md (8–12px)  
  - selected 시 : Primary bg + white text, scale 1.05 애니메이션  
  - 간격 : chip 간 12px, 행 간 16px  

### 3.3 타이포그래피 & 색상 위계
- 메인 질문 : Pretendard Bold 26px, #111827, center  
- 서브 설명 : Medium 16px, #4B5563, center  
- 라벨 : Medium 15px, #374151  
- 플레이스홀더 : Regular 15px, #9CA3AF italic  
- 버튼 텍스트 : Bold 16px, white  

### 3.4 시선 유도 & 그룹화
- 섹션 분리 : 스텝 영역과 입력 영역 사이에 연한 그레이 라인(#E5E7EB) 또는 배경색 차이 (#F9FAFB vs #FFFFFF)  
- 그룹화 : 관련 필드(보험상품 유형 → 치료 구분 → 병원 명)를 하나의 Card로 묶거나 margin으로 구분  
- 입력 필드 간격 : 24px (기존 16px)
