# 사정의 고수 - 신뢰 강화 & 효율화 기능 전략 문서  
안심결제(에스크로) + 표준 사정서 템플릿

**작성일** : 2026년 2월  
**작성자** : EUNJIN (with Grok)  
**목적** : 사용자(의뢰인)의 결제 불안 해소와 고수의 사정서 작성 효율성을 동시에 높여  
앱 전환율·충성도·운영 편의성을 극대화하는 기능 전략 수립

## 1. 기능 개요 및 목표

### 1.1 안심결제(에스크로)
- **설명** : 의뢰인이 수임료를 앱에 예치(에스크로) → 고수가 업무 완료 확인 후 자동 지급  
- **주요 목표**  
  - 사용자 불안 해소 : “고수가 돈 받고 연락 두절” 또는 “부당 청구” 우려 제거  
  - 의뢰 전환율 증가 : 결제 장벽 낮춤 → 실제 의뢰율 20~40% 상승 예상  
  - 신뢰 브랜딩 강화 : “사정의 고수 = 안전한 전문 서비스” 이미지 구축  
- **법적 포지셔닝** : 중개 수수료 없음 → 플랫폼은 “임시 보관 툴” 역할만 (고수 직접 수임료 수령 유지)

### 1.2 표준 사정서 템플릿
- **설명** : 모바일에서 사고 유형·피해 금액·부상 정도 등 핵심 수치 입력 → AI/템플릿 기반 사정서 초안 자동 생성  
- **주요 목표**  
  - 고수 업무 시간 단축 : 사정서 초안 작성 30~60분 → 5~10분 수준으로 축소  
  - 전문성 시각화 : 사용자에게 “초안 미리보기” 제공 → 프로세스 투명성 ↑  
  - 증액률 향상 : 체계적 문서화로 누락 항목 최소화  
  - 프리미엄 수익 가능 : 고도화 버전(AI OCR + 자동 채움) 별도 과금

## 2. 구현 위치 및 사용자 플로우

### 2.1 안심결제(에스크로)
**주요 통합 위치**  
- 의뢰 탭 > 진행 중 > 의뢰 상세 페이지 (가장 중요)  
- 상담 탭 > 채팅 중 “계약 진행” 제안 시  
- 프로필 탭 > 고수 관리자 모드 (지급·환불 관리)

**사용자 플로우 예시**  
1. 의뢰 신청 완료 → “진행 중” 상태  
2. 의뢰 상세 페이지 > “계약 및 결제” 섹션  
   - 수임료 금액 표시 (고수가 설정)  
   - “안심결제(에스크로) 사용” 토글 (기본 ON)  
   - 설명 문구 : “수임료는 앱에 안전하게 예치됩니다. 업무 완료 후 고수에게 지급되며, 불만 시 환불 가능합니다.”  
3. 결제 버튼 → 카카오페이/카드 PG 연동 → 예치 완료  
4. 고수 작업 시작 → 완료 시 “업무 완료 확인” 버튼 (고수 측)  
5. 자동 지급 OR 사용자 “최종 확인” 후 지급  
6. 환불 요청 시 : 사용자 → 고수 승인 → 앱 환불 처리

**고수 측 플로우**  
- 새 의뢰 알림 → 예치 완료 확인  
- 업무 종료 → “완료 확인” 버튼 → 자동 지급 트리거  
- 프로필 탭 > 결제 내역 대시보드 (예치·지급·환불 통계)

### 2.2 표준 사정서 템플릿
**주요 통합 위치**  
- 의뢰 탭 > 신청하기 (폼 마지막 단계)  
- 의뢰 탭 > 진행 중 > 의뢰 상세 페이지  
- 상담 탭 > 채팅 중 “사정서 생성” 버튼  
- 프로필 탭 > 고수 관리자 모드 (템플릿 편집·라이브러리)

**사용자 플로우 예시**  
1. 의뢰 폼 입력 완료 (사고 유형, 피해자 정보, 서류 첨부 등)  
2. “사정서 초안 생성” 버튼 (Primary 색상)  
3. 핵심 입력 화면 (5~7개 필드)  
   - 사고 일시·장소  
   - 피해 금액 (치료비·위자료·휴업손해 등)  
   - 부상 정도 (경상/중상/후유장해 등)  
   - 증액 희망 항목 체크  
4. 생성 → PDF 미리보기 (Flutter PDF 뷰어)  
5. “고수에게 전송” OR “수정” 선택  
6. 고수 채팅으로 초안 공유 → 최종 검토·서명

**고수 측 플로우**  
- 채팅 또는 의뢰 상세에서 초안 수신  
- 템플릿 편집·보완 → 최종 PDF 생성·서명  
- 프로필 탭 > 템플릿 관리 (표준 양식 업데이트, 카테고리별 저장)

## 3. 기술 스택 & 개발 우선순위

### 3.1 안심결제(에스크로)
- **PG 연동** : 카카오페이 에스크로 / 토스페이먼츠 / KG이니시스 (에스크로 지원 PG 선택)  
- **백엔드** : Firebase Functions + Firestore (예치 상태 관리, 지급 트리거)  
- **우선순위** : MVP 핵심 (v1.0~v1.1) – 신뢰가 최우선  
- **예상 개발 기간** : 2~4주 (PG 연동 심사 포함)

### 3.2 표준 사정서 템플릿
- **템플릿 엔진** : Flutter pdf 패키지 + OpenAI/Grok API (텍스트 보완)  
- **저장소** : Firestore (템플릿 JSON + 생성 PDF URL)  
- **우선순위** : v1.5 베타 목표 – 효율성 기능  
- **예상 개발 기간** : 2~3주 (AI 프롬프트 튜닝 포함)

## 4. 법적·운영 리스크 관리

### 공통
- 약관 첫 문단 재강조 : “본 앱은 [고수 이름] 손해사정사의 개인 서비스 제공 도구이며, 어떠한 중개 수수료도 받지 않습니다.”  
- 에스크로 & 템플릿 모두 “고수 직접 서비스 보조 툴”로 명시

### 안심결제 특이사항
- PG사 에스크로 계약 시 “플랫폼 중개 아닌 임시 보관” 구조 확인  
- 환불 프로세스 명확화 (고수 동의 + 증빙 필요)  
- 금융위·금감원 가이드 준수 (보험 관련 에스크로 사례 참고)

### 사정서 템플릿 특이사항
- 모든 생성 문서에 “초안이며 법적 효력 없음. 고수 최종 검토 필수” 워터마크  
- 고수가 서명·날인 후에만 최종본으로 인정

## 5. 예상 효과 & KPI

- **안심결제**  
  - 의뢰 전환율 +25~50%  
  - 사용자 NPS (Net Promoter Score) +15~20pt 상승  
  - 환불률 5% 이내 목표

- **사정서 템플릿**  
  - 고수 1건당 처리 시간 40% 단축  
  - 평균 증액률 10~15% 추가 상승 (문서 완성도 ↑)  
  - 프리미엄 전환율 10% 이상 목표
