# 사정의 고수 - 보험 증권 제출 기능 전략 문서

**작성일** : 2026년 2월  
**작성자** : EUNJIN (with Grok)  
**목적** : 보험증권(보험계약증서) 제출 기능을 앱에 통합하여  
고수가 보장 범위·피보험자·특약·고지의무 위반 여부를 조기에 파악할 수 있게 하고,  
초기 자료 누락을 최소화하여 증액 협상·사정서 작성 효율성을 높임

## 1. 왜 보험 증권 제출 기능이 필요한가?

### 실무적 필요성 (손해사정 현장 기준)
- 자동차보험 : 피보험자·보장 한도·특약(대인·대물·자차·무보험차) 확인 필수  
- 실손의료보험 : 계약 시점·고지의무 대상 질병·면책사항·중복보험 여부 판단 핵심  
- 배상책임·기타 : 보험사별 약관 해석·보상 한도 초과 여부 확인  
- 대부분 보험사 청구 시 **보험증권 사본** 또는 **증권번호 + 가입 내용** 제출 요구  
- 누락 시 : 고수가 후속 요청 → 지연 → 사용자 불만 ↑, 증액 기회 손실  

### 앱 내 이점
- 고수 측 : 초기 단계에서 핵심 정보 확보 → 사정 방향성 빠르게 잡음  
- 사용자 측 : 한 번에 자료 제출 → 반복 입력·전송 최소화  
- 전체 : 의뢰 품질 향상 → 평균 증액률 상승 + 처리 속도 단축

## 2. 구현 위치 및 플로우 (추천: 별도 탭 대신 기존 흐름 통합)

### 추천 방식 : 별도 탭 추가 X → 의뢰 탭 내부 전용 섹션으로 통합
- 이유 : 바텀 네비게이션 과부하 방지, 사용자 흐름(의뢰 신청 → 진행 관리) 유지  
- 주요 위치  
  1. 의뢰 탭 > 신청하기 (폼 단계)  
  2. 의뢰 탭 > 진행 중 > 의뢰 상세 페이지  
  3. 상담 탭 > 채팅 (보완 요청용)

### 2.1 의뢰 신청 폼 내 통합 (초기 제출 최우선)
**단계 배치**  
- 기존 폼 순서 : 사고 유형 → 피보험자/피해자 정보 → 사고 경위 → 부상 정도 → 서류 첨부  
- 삽입 위치 : 사고 유형 선택 후 또는 피보험자 정보 바로 다음 (2~3단계)  
- 새 섹션 이름 : “보험 정보 입력” 또는 “보험증권 제출”

**입력 항목 (필수/선택 구분 명확히)**
- 보험사 선택 (드롭다운)  
  - 옵션 : 삼성생명, 한화생명, DB손해보험, 현대해상, KB손해보험, 삼성화재 등 주요 10개 + “기타”  
- 증권번호 (텍스트 입력)  
  - 플레이스홀더 : “증권번호 입력 (하이픈 포함 가능)”  
  - 자동 포맷팅 : 입력 시 하이픈 자동 추가 (선택)  
- 보험증권 사진/사본 업로드 (카메라 + 갤러리)  
  - 최대 3장 (앞면·뒷면·특약 페이지)  
  - 업로드 가이드 팝업 (필수 노출)  

  - 미리보기 + 재촬영/삭제 버튼  
- “증권 내용 모름” 체크박스  
→ 체크 시 업로드 필수 해제 + “고수가 확인 후 추가 요청드릴게요” 안내 메시지

**UX 최적화**
- 사고 유형이 “실손” 또는 “자동차보험”일 때만 강조 표시 (Required 뱃지)  
- 비회원 : 업로드 불가 → “로그인 후 제출 가능” 팝업  
- 업로드 완료 시 “보험 정보 확인 중” 상태 자동 업데이트

### 2.2 진행 중 상세 페이지 (보완·수정용)
**섹션 구성**
- “보험 증권 자료” 카드  
- 초기 업로드 파일 리스트 (썸네일 + 파일명 + 업로드 날짜)  
- “추가/수정 업로드” 버튼 (Primary 색상)  
- 상태 뱃지 : “미제출” → “제출 완료” → “고수 확인 중” → “완료”  
- 고수 측 : 새 업로드 시 푸시 알림 (“보험증권 업데이트: [의뢰 번호]”)

### 2.3 상담 채팅 연계 (실시간 보완)
- 고수 템플릿 메시지 : “보험증권 사진 보내주시면 더 정확히 도와드릴 수 있어요.”  
- 사용자 : 채팅 하단 파일 첨부 아이콘 클릭 → 바로 업로드  
- AI 챗봇 자동 제안 : “보험증권이 필요해 보이네요. 첨부해주세요!”

## 3. 기술 스택 & 개발 우선순위

### 기술 스택
- 프론트 : Flutter (image_picker, file_picker, pdf 뷰어 패키지)  
- 백엔드 : Firebase Storage (파일 저장) + Firestore (증권번호·파일 URL·상태 저장)  
- 보안 : 업로드 파일 AES 암호화 + Firestore Security Rules (고수만 읽기 가능)

### 개발 우선순위 (MVP 기준)
1. 의뢰 폼 내 업로드 버튼 + 가이드 팝업 (1~2주)  
2. 진행 중 상세 페이지 섹션 + 상태 뱃지 연동 (1주)  
3. 채팅 연계 + 푸시 알림 (1주)  
총 예상 : 3~4주

### 예상 비용
- Flutter 개발자 1명 기준 : 400~800만 원  
- Firebase 추가 비용 : 월 5~20만 원 (스토리지 사용량)

## 4. 보안·법적·운영 대응

- **보안**  
- 업로드 시 “개인정보 보호” 강조 문구 + 블라인드 추천 가이드 이미지  
- 파일 접근 : 고수만 가능 (사용자 본인도 자신의 파일만 볼 수 있게)  
- **법적 안전성**  
- 약관 재강조 : “증권 자료는 고수 직접 사정 보조용이며, 중개 목적 아님”  
- “개인정보는 암호화 저장되며 제3자에게 공유되지 않습니다” 문구 필수  
- **운영 팁**  
- 업로드 실패 시 재시도 버튼 + 에러 메시지 명확히  
- 최대 파일 크기 10MB, 형식 JPG/PNG/PDF 제한  
- 고수 대시보드 : 모든 의뢰의 증권 상태 한눈에 확인 가능

## 5. 예상 효과 & KPI

- 초기 의뢰 시 증권 제출률 70% 이상 목표  
- 자료 누락으로 인한 후속 요청 40~50% 감소  
- 평균 처리 속도 15~25% 단축  
- 사용자 만족도 (의뢰 후 설문) +10pt 이상 상승 예상
