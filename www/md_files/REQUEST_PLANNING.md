📜 사정의 고수 - 3단계 의뢰 신청 폼 구현 프롬프트
목표: React(또는 Flutter/React Native)를 사용하여 사용자의 선택에 따라 입력 필드가 동적으로 변하는 3단계 의뢰 신청 폼을 구현한다.

1. 공통 레이아웃 (Layout)
상단 스텝 바: 1단계 사고, 2단계 보험사, 3단계 상황으로 구성된 인디케이터. 현재 단계는 강조 표시.

메인 타이틀: "손해사정사 선임이 가능할까? 신청만 하면 선임가능 여부를 확인해 연락드려요".

하단 버튼: 이전, 다음 (마지막 단계는 신청하기) 버튼 배치.

2. [1단계] 사고 정보 (Step 1: Accident)
사용자가 선택한 '보험상품 유형'에 따라 아래 필드들을 동적으로 렌더링한다.

A. 기본 선택 (Dropdown)
보험상품 유형 선택: [실손, 배상책임/재물, 교통사고, 후유장해, 진단/수술비/기타] 중 택 1.

B. 유형별 가변 입력 UI (Conditional Rendering)
[실손] 선택 시:

치료 구분 (다중 선택 체크박스): 백내장, 신경성형술, 도수/체외충격파, 남성질환, 여성질환, 어린이, 무릎치료, 고지의무, 기타.

병원 명 (텍스트 입력): "병원 지역/명 입력" 플레이스홀더.

청구 상황 (가로형 선택 버튼): [보험금 청구 전, 보험금 접수, 현장조사 안내, 심사 중, 보험금 거절].

[배상책임/재물] 선택 시:

치료 구분 (다중 선택 체크박스): 대인배상, 대물배상, 누수, 화재/기타.

본인 구분 (가로형 선택 버튼): [피해자, 피보험자(보험 가입자)].

지역 명 (텍스트 입력): "사고 지역 입력" 플레이스홀더.

[교통사고] 선택 시:

치료 구분 (단일 체크박스): 교통사고.

병원 명 (텍스트 입력): "병원 지역/명 입력" 플레이스홀더.

[후유장해] 선택 시:

치료 구분 (다중 선택 체크박스): 질병후유장해, 상해후유장해.

병원 명 (텍스트 입력): "병원 지역/명 입력" 플레이스홀더.

[진단/수술비/기타] 선택 시:

치료 구분 (다중 선택 체크박스): 암진단, 사망진단, 뇌혈관 진단, 심장질환, 수술비/기타.

병원 명 (텍스트 입력): "병원 지역/명 입력" 플레이스홀더.

청구 상황 (가로형 선택 버튼): 실손과 동일한 5개 옵션.

3. [2단계] 보험사 정보 (Step 2: Insurance)
가입된 보험의 종류와 상세 보험사를 선택한다.

A. 보험 금액 및 종류
보험 금액 (Input): 우측에 '원' 단위 표시. 옆에 '모름' 체크박스 배치.

보험 종류 (Dropdown): [손해보험, 생명보험] 중 택 1.

B. 보험사 리스트 (Grid Layout)
[손해보험] 선택 시 노출: 현대해상, 흥국화재, 삼성화재, DB손해보험, 롯데손해보험, 메리츠화재, KB손해보험, 한화손해보험, 기타.

[생명보험] 선택 시 노출: 교보생명, 한화생명, 삼성생명, NH농협생명, 흥국생명, ABL생명, 기타.

UI: 4열 그리드 형태의 체크박스 버튼으로 구현.

4. [3단계] 상세 상담 신청 (Step 3: Situation)
최종 개인 정보와 내용을 입력한다.

이름 (Input): "성함을 입력해주세요".

연락처 (Input): "010-0000-0000" (숫자 키패드 권장).

상담 내용 (Textarea): "상담 내용을 입력해주세요". 우측 하단에 글자 수 표시 (0/500자).

연락 가능 시간대 (Input): "전화 받기 편한 시간".

최종 액션: 신청하기 버튼 클릭 시 전체 데이터를 JSON 객체로 취합하여 전송.

5. 기술적 요구사항
상태 관리: 전체 폼 데이터는 하나의 객체로 관리하며 단계 이동 시 데이터가 유지되어야 한다.

유효성 검사: 각 단계의 필수 항목(보험유형, 연락처 등)이 입력되지 않으면 '다음' 또는 '신청하기' 버튼을 비활성화한다.

스타일링: 깨끗한 화이트 배경에 Primary 컬러(#3B82F6)를 포인트로 사용하며, 이미지와 동일한 간격과 폰트 크기를 유지한다.
gpt 스러운 느낌은 배제하고 세련된 느낌으로 구성되도록 한다.
