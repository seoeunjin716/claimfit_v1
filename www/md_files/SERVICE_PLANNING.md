# 사정의 고수 - 모바일 프론트엔드 스펙 & 와이어프레임 (Mobile First)

**프로젝트명** : 사정의 고수  
**슬로건** : 한 명의 고수가 당신의 보상을 책임집니다  
**타겟** : 의뢰인 (주 사용자) + 고수 손해사정사 (앱 운영자/관리자)  
**앱 구조** : 한 명의 손해사정사 독점 운영 구조 (개인 브랜딩 툴)  
**우선 개발 대상** : Mobile (iOS / Android WebView 또는 React Native / Flutter 등 하이브리드)  
**디자인 원칙**  
- Mobile First : 375px × 812px (iPhone 13/14 기준)부터 시작  
- Breakpoints 예시  
  - Mobile : ~ 480px  
  - Tablet : 481px ~ 1024px (나중 확장)  
  - Desktop : 1025px ~ (웹 버전은 v2 이후 고려)  
- Touch 우선 : 버튼/터치 영역 최소 48×48px 권장  
- Safe Area 고려 (iOS notch, Android gesture bar)  
- Dark Mode 지원 준비 (시스템 따라 자동 적용)

## 1. 전체 앱 구조 (Bottom Tab + Stack Navigation)

### Bottom Tab Navigator (메인 내비게이션)
| 탭       | 아이콘 (예시)     | 주요 화면                          | 사용자 |
|----------|-------------------|------------------------------------|--------|
| 홈       | house / star      | 고수 프로필 카드 + 성공 사례 + 빠른 상담 시작 | 의뢰인 |
| 상담     | message-square    | AI 챗봇 + 고수 1:1 채팅 통합 공간         | 의뢰인 |
| 의뢰     | document-plus     | 의뢰 신청 폼 + 진행 중/완료 의뢰 리스트    | 의뢰인 |
| 사례     | folder / toolbox  | 성공 사례 라이브러리 + 체크리스트 & 가이드  | 의뢰인 |
| 프로필   | user-circle       | 사용자 정보 + 고수 정보 관리              | 의뢰인 + 고수 |

- 기본 : 의뢰인 중심 UI (고수는 프로필 탭에서 관리자 모드로 접근)  
- 고수 관리자 모드 : 프로필 탭에서 내부 통계, 케이스 업로드, 프로필 편집 등

## 2. 핵심 화면 목록 (Mobile First 순서)

### 2.1 앱 실행 플로우
**전체 오픈 플로우**  
앱 실행 → 스플래시 화면 (로고 + "사정의 고수" + 로딩 1~2초) → 로그인 화면 → 성공 시 온보딩 (최초 1회) → 홈 탭

### 2.2 스플래시 화면
- 앱 로고 + "사정의 고수" 텍스트  
- 로딩 애니메이션 (1~2초)  
- 자동 로그인 체크 (토큰 있으면 홈으로, 없으면 로그인 화면으로)

### 2.3 로그인 화면
**위치** : 앱 첫 실행 시 무조건 로그인 화면부터 표시  
**구성**  
- 상단 : 앱 로고 + 슬로건 ("한 명의 고수가 당신의 보상을 책임집니다")  
- 중앙 : 소셜 로그인 버튼 (큰 버튼 3~4개, 순서 고정)  
  - **카카오로 로그인** (1위 버튼, 가장 크게 표시, 노란색)  
  - 네이버로 로그인 (초록색)  
  - Google로 로그인 (파랑/흰색)  
  - (선택) 이메일/휴대폰 번호로 로그인 (SMS 인증)  
- 하단  
  - "로그인 없이 둘러보기" 버튼 (비회원 허용 범위: 사례 보기 + 체크리스트 입력까지만, 의뢰/채팅 시작 시 무조건 로그인 팝업 강제)  
  - "회원가입" 링크 (소셜 로그인 시 자동 가입 처리)  
  - 개인정보 처리방침 + 이용약관 링크 (웹뷰로 열기, 이용약관에 "중개 플랫폼 아님" 명시 필수)  
- 로딩 중 : "고수가 기다리고 있어요..." 애니메이션

**보안 & UX**  
- 소셜 로그인 시 최소 정보만 요청 (이름, 이메일, 연락처)  
- 자동 로그인 유지 : Secure Storage에 토큰 저장  
- **비회원 허용 범위 명확화**  
  - 허용 : 사례 보기 + 체크리스트 입력까지만 가능  
  - 제한 : 의뢰/채팅 시작 시 무조건 로그인 팝업 강제 표시 ("고수와 상담하려면 로그인해주세요")

### 2.4 온보딩 화면 (최초 로그인 시 1회만)
- 스와이프 가능한 2~3장  
  1. "고수의 실제 성공 사례를 먼저 보세요"  
  2. "간단 상담으로 시작하고 고수와 연결"  
  3. "의뢰 신청 → 빠른 계약 → 보상 증액"  
- 마지막 : "시작하기" 버튼 → 홈 탭으로 이동

### 2.5 홈 화면 (메인 랜딩 페이지)
**역할** : 사용자 첫 인상 + 즉시 행동 유도 (상담/의뢰로 연결)  
**구성**  
- 상단 : 고수 프로필 카드 (사진 + 증액률 + 성공 케이스 수 강조)  
  - **법적 안전성 강화** : "직접 수임 · 개인 운영" 뱃지 필수 표시 (오인 방지)  
- 중앙 : "빠른 상담 시작" 큰 버튼 (Primary 색상, 48px+ 높이)  
- 성공 사례 미리보기 슬라이드 (가로 스크롤, 썸네일 3~5개)  
- 프로모션 배너 ("고수가 책임집니다!" 등)  
- 하단 : 스크롤 가능한 추가 콘텐츠 (신뢰 지표, 리뷰 요약 등)

### 2.6 상담 화면 (AI 챗봇 + 고수 1:1 채팅 통합)
**역할** : 앱의 핵심 기능 (대부분의 사용자 여기가 제일 많이 머무름)  
**구성**  
- 기본 모드 : AI 챗봇 (간단 질문 자동 응답)  
- 복잡한 질문 시 : "고수 연결" 버튼 표시 → 고수에게 푸시/채팅 넘김  
  - **상태 표시 필수** : AI 모드 → 고수 연결 전환 시 "고수가 직접 확인 중입니다" 메시지 표시 (사용자 불안 완화)  
- 채팅 UI (카카오톡 스타일)  
  - 사진/파일 업로드 버튼  
    - **보안 안내** : 파일 업로드 시 "개인정보 암호화 저장" 안내 문구 표시 (ISMS-P 준비 중이라면 강조)  
  - 음성 녹음 버튼 (길게 누르기)  
  - 전자위임장 요청 버튼 (템플릿 자동 생성)  
  - "진행 방향 제안" 템플릿 제안  
- 우측 상단 : 의뢰 정보 요약 (접기/펼치기)

### 2.7 의뢰 화면 (의뢰 신청 및 관리)
**역할** : 실제 돈 버는 흐름 (의뢰 → 계약 → 수임)  
**구성**  
- 상단 탭 : 신청하기 / 진행 중 / 완료  
- **신청하기 탭** : 의뢰 폼 (5분 이내 완료 목표)  
  - 단계별 폼 (Progress Bar)  
    1. 사고 유형 선택  
    2. 사고 장소 (텍스트 + 지도 핀)  
    3. 사고 경위 (짧은 텍스트)  
    4. 부상 정도 선택  
    5. 피해자 정보 (생년월일, 연락처)  
    6. 사진/서류 첨부 (최대 10장)  
  - 마지막 : "고수에게 의뢰하기" 버튼  
- **진행 중 탭** : 상태별 리스트 (상담 중 → 계약 → 진행)  
  - 각 의뢰 카드 : 상태 뱃지 + 전자계약/결제 버튼  
- **완료 탭** : 완료된 의뢰 내역  
- 고수 입장 : 새 의뢰 푸시 알림 + 관리 대시보드 연동

### 2.8 사례 화면 (성공 사례 라이브러리 + 체크리스트)
**역할** : 신뢰 쌓기 + 자가 진단 후 자연스러운 의뢰 유도  
**구성**  
- 상단 : 필터링 옵션 (사고 유형, 증액률, 연도 등)  
- 메인 : 성공 사례 카드 리스트 (스크롤)  
  - 케이스 카드 구성  
    - 사고 유형 뱃지  
    - 증액 전/후 금액 비교  
    - 쟁점 요약  
    - 익명화된 썸네일 (선택)  
- 하단 섹션 : 보상 항목 체크리스트 & 가이드  
  - 체크리스트 입력 폼  
  - 항목별 가이드 (툴팁/모달)  
  - "상담 받기" 버튼 (상담 화면으로 연결)

### 2.9 프로필 화면 (사용자 + 고수 정보 관리)
**역할** : 개인화 + 운영 편의 (고수가 앱 관리자로 콘텐츠 업데이트)  
**의뢰인 모드**  
- 내 정보 (이름, 연락처, 생년월일)  
- 과거 의뢰 내역 (링크)  
- 알림 설정  
- 로그아웃 버튼  
**고수 관리자 모드** (고수 계정으로 로그인 시)  
- 프로필 편집 (경력, 성공률 업데이트)  
- 케이스 업로드 (성공 사례 추가)  
- 내부 메모/통계 (의뢰 수, 증액 총액 등)  
- 콘텐츠 관리 (배너, 프로모션 등)

## 5. 개발 우선순위 & 수익 모델

### MVP 개발 우선순위
1. **로그인** (소셜 로그인: 카카오, 네이버, Google)  
2. **AI 챗봇** (초기 필터링 및 자동 응답)  
3. **의뢰 폼** (5분 이내 완료 가능한 단계별 폼)  
4. **채팅** (고수 1:1 채팅 통합)  
5. **홈 화면** (고수 프로필 + 빠른 상담 시작)  
6. **사례 화면** (성공 사례 라이브러리 + 체크리스트)

### 수익 모델
- **주요 수익** : 고수가 직접 수임료 받음 (앱은 중개 수수료 없음)  
- **앱 다운로드** : 무료  
- **프리미엄 기능** (추가 수익 가능)  
  - AI 상세 분석 리포트  
  - 우선 상담 대기열  
  - 추가 케이스 라이브러리 접근

### 법적 안전성
- 광고형 또는 개인 서비스 앱으로 운영  
- 중개형 불법 소지 리스크 회피  
- 직접 수임료 모델로 법적 안전성 확보  
- 상세 보완 사항은 "6. 보완/주의할 점" 섹션 참조

## 6. 보완/주의할 점 (구체적 제안)

### 법적 안전성 추가 강화 포인트
- **이용약관 명시 필수**  
  - 이용약관 첫 문단: "본 앱은 [고수 이름] 손해사정사의 개인 서비스 제공 도구로, 어떠한 중개 수수료도 받지 않습니다."  
  - 앱 설명/슬로건에도 "중개 플랫폼 아님" 명시  
- **홈 화면 고수 카드**  
  - "직접 수임 · 개인 운영" 뱃지 필수 표시 (오인 방지로 금감원/금융위 기준 더 안전)

### 로그인 화면 보완
- **비회원 범위 명확화**  
  - 허용: 사례 보기 + 체크리스트 입력까지만 가능  
  - 제한: 의뢰/채팅 시작 시 무조건 로그인 팝업 강제 표시  
- **소셜 로그인 버튼 순서**  
  - 카카오가 1위 버튼으로 가장 크게 표시 (한국 사용자 80% 이상 사용)

### 상담 화면 보완
- **AI → 고수 연결 전환 시**  
  - "고수가 직접 확인 중입니다" 상태 표시 필수 (사용자 불안 완화)  
- **파일 업로드 시**  
  - "개인정보 암호화 저장" 안내 문구 추가 (ISMS-P 준비 중이라면 강조)
