# 사정의 고수 - 모바일 프론트엔드 스펙 & 와이어프레임 (Mobile First, 로그인 우선 + AI 챗봇 채팅 통합)

**프로젝트명** : 사정의 고수  
**타겟** : 손해사정사 (주 사용자) + 의뢰인 (보조)  
**우선 개발 대상** : Mobile (iOS / Android WebView 또는 React Native / Flutter 등 하이브리드)  
**디자인 원칙**  
- Mobile First : 375px × 812px (iPhone 13/14 기준)부터 시작  
- Breakpoints 예시 : Mobile ~480px → Tablet 481~1024px (후순위)  
- Touch 우선 : 버튼/터치 영역 최소 48×48px (권장 56px 이상)  
- Safe Area 고려 (notch, gesture bar)  
- Dark Mode 지원 준비  
- **업데이트 포인트** : AI 챗봇을 별도 탭 대신 채팅 탭에 통합 (채팅 목록에 AI 봇 추가, 기존 채팅과 병행). 탭 수 5개 유지 (홈, 의뢰, 채팅, 커뮤니티, 마이). 각 탭 내부 기능 구체화 (MVP 중심으로 세분화, 확장성 고려).  
- **왜 AI를 채팅에 통합?** : 사용자 흐름 자연스럽게 (채팅처럼 대화 → AI 상담 → 사정사 채팅 전환). 탭 과부하 방지, 기능 집중.

## 1. 전체 앱 구조 (Bottom Tab + Stack Navigation, 로그인 우선)

### 초기 흐름 (로그인 강제)
1. **스플래시 화면** (로고 + 로딩, 1.5~2초)  
2. **로그인 화면** (항상 첫 화면)  
3. 로그인 성공 → **홈 화면** (기능 허브 대시보드)  
4. Bottom Tab Navigator 활성화 (홈 포함)

### Bottom Tab Navigator (메인 내비게이션, 5개 유지)
| 탭       | 아이콘 (예시)     | 주요 화면                          | 비고 (구체화된 내부 기능) |
|----------|-------------------|------------------------------------|---------------------------|
| 홈       | house-fill        | 기능 허브 (대시보드 스타일)        | Quick Cards로 모든 기능 접근, 개인화 추천 (eunjin님처럼 사용자 맞춤) |
| 의뢰     | clipboard-list    | 의뢰 풀 / 진행중                   | 필터링, 선택 매칭, 상세 미리보기 확장 |
| 채팅     | message-square-fill | 1:1 채팅 목록 + AI 챗봇 통합      | AI 봇 채팅방 포함, 파일 공유, 전자계약 |
| 커뮤니티 | users-fill        | 사례 공유 / Q&A / 네트워킹         | 게시판, 멘토링 매칭, 고수 뱃지 시스템 |
| 마이     | user-circle-fill  | 프로필 / 설정 / 수익 내역         | 프로필 강화, 알림/평판 관리, 확장 기능 |

- **사정사 모드** : 모든 탭 풀 액세스.  
- **의뢰인 모드** : 의뢰/채팅 중심, 커뮤니티 제한 (읽기 전용).  
- **탭 5개 유지 이유** : UX 단순화. 내부 기능 늘리기 위해 서브 메뉴/모달/스택 네비 활용 (예: 채팅 내 AI 전환).

## 2. 핵심 화면 목록 (Mobile First 순서, 기능 구체화)

### 2.1 스플래시 & 로그인 (최우선, 기존 유지)
- **스플래시** : 로고 중앙 + "사정의 고수" 텍스트 + 로딩 인디케이터.  
- **로그인 화면** (전체 화면)  
  - 상단: 앱 로고 + 슬로건.  
  - 중간: 소셜 버튼 (카카오, 네이버, 구글, 전화번호).  
  - 하단: 회원가입 링크.  
  - **구체화** : 로그인 후 모드 선택 모달 ("사정사입니까? 의뢰인입니까?" → 프로필 기반 자동화 가능).

### 2.2 홈 화면 (기능 허브 대시보드, 기존 유지 + 구체화)
- **레이아웃** : 헤더 + Quick Action Cards + 추천 섹션 + FAB.  
- **Quick Action Cards (구체화)**  
  - 새 의뢰 확인: 실시간 푸시 카운트, 필터 적용 (지역/유형).  
  - AI 상담 시작: 채팅 탭으로 직접 연결 (AI 봇 방 오픈).  
  - 진행중 의뢰: 상태별 필터 (대기/협상/완료), 클릭 시 채팅 이동.  
  - 커뮤니티 바로가기: 핫 토픽 미리보기 (최신 Q&A).  
- **추천 섹션 (구체화)**  
  - AI 기반 추천: Gemini로 "eunjin님, 자동차 사고 의뢰 추천" (개인화).  
  - 고수 랭킹: 활동 포인트 기반 (성공 케이스 + 도움 횟수).  
- **FAB** : "+" → 새 의뢰 등록 (의뢰인) 또는 의뢰 검색 (사정사).

### 2.3 의뢰 화면 (구체화: 사정사 주도 매칭 강조)
- **메인 리스트** : 스크롤 카드 (새 의뢰 / 내 의뢰).  
  - **기능 구체화**  
    - 필터: 지역, 유형, 긴급도, AI 예상 보상액 (Gemini 통합).  
    - 카드 상세: 기본 정보 (사고 장소/경위/부상/생년월일/연락처) + "미리보기" 버튼 (AI 분석 결과).  
    - 선택 매칭: "수락" 버튼 (선착순/입찰) → 채팅 자동 생성.  
    - 검색 바: 키워드 검색 (e.g., "자동차 사고").  
- **의뢰 등록 (의뢰인 서브 화면)** : 단계 폼 + AI 도움 (Gemini로 자동 완성 제안).

### 2.4 채팅 화면 (AI 챗봇 통합 핵심, 구체화)
- **채팅 목록** : 스크롤 리스트 (1:1 채팅방 + AI 봇 방 상단 고정).  
  - **AI 챗봇 통합 (제안 포인트)**  
    - 별도 AI 방: "AI 상담 봇" 이름으로 고정 (채팅 목록 최상단).  
    - 채팅 UI: 텍스트 입력 + 사진 업로드 + 음성 녹음.  
    - Gemini 연동: 사진 → 손상 분석/예상 보상액 (e.g., "중증, 500~800만 원").  
    - 전환: AI 상담 중 "사정사 연결" 버튼 → 의뢰 매칭.  
  - **기타 채팅 기능 구체화**  
    - 파일 공유: 사진/PDF 업로드 + 다운로드.  
    - 전자계약: "위임장 생성" 버튼 → 템플릿 + 전자서명 (카카오페이 연동).  
    - 알림: 실시간 푸시 (새 메시지/서류 마감).  
    - 내부 메모: 사정사만 보는 노트 (e.g., "의뢰인 서류 지연").  
- **왜 채팅에 통합?** : 사용자 "채팅처럼" 느끼게, 기능 중복 줄임. 내부 늘리기: 모달로 AI 고급 옵션 (e.g., 판례 검색).

### 2.5 커뮤니티 화면 (구체화: 성장 중심)
- **탭 내 서브 탭** : 전체 / 성공사례 / Q&A / 모임.  
  - **기능 구체화**  
    - 게시판: 익명 포스트 + 태그 (쟁점별) + 댓글.  
    - Q&A: "질문 올리기" + 답변 랭킹 (고수 뱃지 우선).  
    - 네트워킹: 지역 모임 알림 + 1:1 멘토링 신청 (매칭 알고리즘).  
    - 고수 뱃지: 자동 부여 (활동량/도움 횟수) → 프로필 표시 + 의뢰 우선 노출.  
    - 검색: 키워드 + AI 요약 (Gemini로 "이 쟁점 요약해").  
- **확장 아이디어** : 오프라인 세미나 예약 (나중 추가).

### 2.6 마이 화면 (구체화: 관리 중심)
- **섹션** : 프로필 / 수익 / 설정 / 평판.  
  - **기능 구체화**  
    - 프로필: 편집 (전문 분야/경력/성공 사례 업로드) + 뱃지 갤러리.  
    - 수익 내역: 착수금/성공보수 + 자동 정산 (보험금 확인).  
    - 설정: 알림 필터 / 다크모드 / 계정 관리.  
    - 평판: 내부 메모 + 상호 평가 (의뢰인/사정사).  
    - AI 이력: 과거 상담 로그 + 분석 정확도 (피드백 시스템).
